# План запуска и настройки проекта

Ниже приведен контрольный список действий для первого запуска системы.

## ✅ Шаг 1: Подготовка окружения
- [ ] Установить **Node.js LTS** (например, v20.x).
- [ ] Установить **SQL Server Express** или Docker-контейнер с MSSQL.
- [ ] Создать пустую базу данных (например, `trpo`).

## ✅ Шаг 2: Настройка Бэкенда (`/backend`)
- [ ] `npm install` (установка зависимостей NestJS и Prisma).
- [ ] Настроить файл `.env`:
  - Ключ `DATABASE_URL` (строка подключения к SQL Server).
  - Ключ `JWT_SECRET` (произвольная строка для шифрования токенов).
- [ ] Выполнить синхронизацию схемы: `npx prisma db push`.
- [ ] Генерация клиента: `npx prisma generate`.
- [ ] Первоначальное наполнение тестовыми данными: `npm run seed`.

## ✅ Шаг 3: Настройка Фронтенда (`/frontend`)
- [ ] `npm install` (установка **Electron** и зависимостей).
- [ ] Убедиться, что в `package.json` скрипт `"start"` указывает на `electron .`.

## ✅ Шаг 4: Порядок запуска
1. Сначала запустить бэкенд: `npm run start:dev` в папке `/backend`.
2. После успешного запуска бэкенда, запустить фронтенд: `npm start` в папке `/frontend`.

---

## 📦 Основные используемые библиотеки

### Бэкенд (backend)
- **@nestjs/core**: Основной фреймворк.
- **prisma**: ORM для работы с SQL Server.
- **passport-jwt**: Обеспечение авторизации по токенам.
- **bcrypt**: Шифрование паролей.
- **class-validator**: Валидация входных данных.

### Фронтенд (frontend)
- **electron**: Среда для запуска десктопного приложения.
- **font-awesome**: Иконки интерфейса.
- **vanilla-js**: Чистый JavaScript для логики без лишних весов.
