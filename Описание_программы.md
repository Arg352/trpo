## Основная часть

### Общие сведения
*   **Обозначение и наименование программы:** Программный комплекс «SmartPicker» (Warehouse Micromanagement System).
*   **Программное обеспечение, необходимое для функционирования:**
    *   Среда исполнения: Node.js v18.x или выше.
    *   База данных: Microsoft SQL Server 2019+.
    *   Оболочка клиента: Electron Framework.
*   **Языки программирования:**
    *   TypeScript (серверная логика на NestJS).
    *   JavaScript (клиентская логика Electron).
    *   CSS/HTML (пользовательский интерфейс).
    *   SQL (структура БД через Prisma ORM).

### Функциональное назначение
*   **Классы решаемых задач:** Оперативное управление складскими процессами, мониторинг работы сотрудников, распределение приоритетных заказов, навигация сборщиков по складским ячейкам.
*   **Назначение программы:** Оптимизация процесса сборки заказов на складе за счет четкой иерархии (Менеджер -> Бригадир -> Сборщик) и автоматизации назначения задач.
*   **Сведения о функциональных ограничениях:**
    *   Требуется активное сетевое соединение с сервером БД.
    *   Необходима предварительная регистрация пользователей администратором.
    *   Ограничение очереди заказов для рабочего (максимум 2 параллельно).

### Описание логической структуры
*   **Алгоритм программы:**
    1.  Инициализация соединения с БД и запуск HTTP-сервера.
    2.  Аутентификация пользователя и получение роли.
    3.  Для роли **Старший менеджер**: выбор заказов со статусом 'new' и назначение их на команды (Teams).
    4.  Для роли **Бригадир**: выбор заказов команды и назначение их свободным сотрудникам.
    5.  Для роли **Сборщик**: выполнение активного заказа по шагам, отметка товаров в ячейках.
    6.  Закрытие заказа с обновлением статусов в реальном времени.
*   **Используемые методы:**
    *   REST API для взаимодействия фронтенда и бэкенда.
    *   JWT (JSON Web Tokens) для обеспечения безопасности сессий.
    *   Prisma ORM для безопасного и типизированного доступа к данным SQL Server.
*   **Структура программы:**
    *   `backend/src/orders`: Обработка логики заказов и их статусов.
    *   `backend/src/users`: Управление профилями и ролями.
    *   `backend/src/teams`: Логика формирования бригад.
    *   `frontend/js/index.js`: Основной контроллер интерфейса Electron.
*   **Связи с другими программами:** Приложение использует драйвер `tedious` для работы с SQL Server и встроенные API Electron для управления окнами ОС.

### Используемые технические средства
*   Процессор: не менее 2 ядер, частота от 2.0 ГГц.
*   Оперативная память: не менее 4 ГБ.
*   Свободное место: 200 МБ для приложения + объем БД.
*   Периферия: Мышь, клавиатура, монитор с разрешением от 1280x720.

### Вызов и загрузка
*   **Способ вызова:**
    *   Сервер: `npm run start:dev` из папки `backend`.
    *   Клиент: `npm start` из папки `frontend`.
*   **Входные точки:**
    *   Бэкенд: `backend/src/main.ts`.
    *   Фронтенд: `frontend/main.js`.

### Входные данные
*   **Характер и организация:** Учетные данные (логин/пароль), реестр заказов во внешнем или внутреннем формате JSON, справочник товаров.
*   **Формат и кодирование:** Текстовые данные в кодировке UTF-8, передача через JSON по протоколу HTTP/HTTPS.

### Выходные данные
*   **Характер и организация:** Статистика производительности (KPI), обновленные статусы товаров в БД, отчеты о сборке.
*   **Формат и кодирование:** JSON-ответы API, записи в таблицах SQL Server, визуальные уведомления в интерфейсе.
